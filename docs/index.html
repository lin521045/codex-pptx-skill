pptx-codex

$ npx skills add https://github.com/lin521045/codex-pptx-skill --skill pptx-codex

摘要

创建、编辑、读取和处理 PowerPoint 演示文稿，并提供设计指导与质量保证流程。

  • 支持三条主要工作流：读取和提取现有 .pptx 文本、通过模板解包与重打包编辑演示文稿、使用 PptxGenJS 从零创建 deck
  • 提供 10 组精选配色和字体搭配，避免通用 AI 幻灯片风格，并给出版式模式、间距与对比度规则
  • 提供强制 QA 流程：占位符检查、图片渲染、视觉复核与 Codex 适配的二轮检查流程
  • 提供文本提取、缩略图生成、XML 解包、复制 slide、清理、重打包、PDF/图片导出等工具

从零创建

完整说明见 pptxgenjs.md。

当没有模板或参考 deck 时，使用 PptxGenJS 从零创建。

设计思路

不要做无聊的 PPT。 白底 bullet 页不适合答辩、汇报或路演。每一页都应该至少有一个视觉锚点。

开始前

  • 选择与内容高度相关的主配色，而不是默认蓝色
  • 让一个颜色占主导视觉权重，另外 1-2 个做辅助
  • 确定深浅结构：深色封面/结尾 + 浅色正文，或整套深色
  • 为整套 deck 选定一个反复出现的视觉母题

每页建议

每一页都需要至少一个视觉元素，例如图片、图表、图标、形状、指标卡片或时间线。

布局

  • 双栏布局
  • 图标 + 文本行
  • 2x2 / 2x3 卡片网格
  • 半出血图片

数据展示

  • 大数字指标
  • 对比页
  • 时间线 / 流程页

元素建议字号
幻灯片标题36-44pt 粗体
分区标题20-24pt 粗体
正文14-16pt
注释10-12pt

间距

  • 0.5" 最小边距
  • 0.3-0.5" 内容块间距
  • 保留呼吸感，不要塞满每一寸画布

避免

  • 整套重复同一种布局
  • 正文居中排版
  • 标题和正文缺乏尺寸对比
  • 默认蓝白配色
  • 标题下装饰线
  • 纯文字页
  • 低对比文字和图标

QA（必须）

默认假设第一页导出一定有问题。 QA 不是确认流程，而是查错流程。

内容 QA

python skills/pptx-codex/scripts/extract_text.py output.pptx
python skills/pptx-codex/scripts/check_placeholders.py output.pptx

视觉 QA

将幻灯片渲染为图片后，检查：

  • 元素重叠
  • 文本裁切或溢出
  • 页边距不足
  • 卡片间距过近
  • 左右不对齐
  • 低对比文字/图标
  • 占位符残留

Codex 版说明：只有用户明确允许多代理时，才使用子代理做第二轮审查；否则执行双轮人工复核。

验证循环

  1. 生成 slides
  2. 转图片
  3. 列出问题
  4. 修复问题
  5. 复查受影响页面

转换为图片

python skills/pptx-codex/scripts/office/soffice.py --headless --convert-to pdf output.pptx
python skills/pptx-codex/scripts/office/render.py output.pptx rendered/

也可以直接用：

python skills/pptx-codex/scripts/thumbnail.py output.pptx

依赖

  • pip install "markitdown[pptx]"：可选文本抽取
  • pip install Pillow：缩略图拼板
  • pip install PyMuPDF：PDF 栅格化
  • npm install pptxgenjs：从零创建
  • LibreOffice soffice：PDF 转换
  • Microsoft PowerPoint（Windows，可选）：PDF 导出和高保真渲染
