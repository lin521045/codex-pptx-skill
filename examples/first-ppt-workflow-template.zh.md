# 第一份示例 PPT 工作流模板

这是一份给 Codex 直接使用的首轮工作流模板。适合“我有一份 brief，希望你帮我做出第一版 PPT”的场景。

## 使用方法

1. 先准备一份 brief，可参考 `source-brief-template.zh.md`。
2. 把 brief 内容贴给 Codex。
3. 再附上下面这段任务模板。

## 模板任务

```text
使用 $pptx-codex 完成这份 PPT 任务。

目标：
- 基于我提供的 brief 或材料，生成一份结构完整、视觉风格明确的 PPT 初稿。

请按下面流程工作：
1. 先判断应该走“模板编辑”还是“从零创建”。
2. 如果我提供了现成 .pptx，请先抽取文本、查看布局，再决定是否沿用模板。
3. 如果没有合适模板，请使用 PptxGenJS 从零生成。
4. 先给出建议页结构和视觉方向，再开始实际生成。
5. 每一页都不要做成普通白底 bullet 页，要尽量使用卡片、对比、时间线、图像区、指标区等结构。
6. 生成后执行 QA：
   - 检查是否有 placeholder 残留
   - 检查是否有拥挤、重叠、对比不足、信息层级混乱
   - 至少进行一轮修复和复核

输出要求：
- 给出建议目录
- 给出配色和字体方向
- 说明每页的布局类型
- 产出可继续迭代的 deck 文件或生成脚本
- 最后说明你做了哪些 QA 检查
```

## 建议搭配

建议一起提供：

- `source-brief-template.zh.md` 填好的内容
- 旧版 deck 或品牌模板
- 必须出现的数据、图表或结论

## 适合的任务类型

- 答辩 PPT
- 汇报 PPT
- 路演 deck
- 研究结果展示
- 产品方案讲解
